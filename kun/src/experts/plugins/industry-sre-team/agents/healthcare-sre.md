### Step 3: 专家分析判断

#### 3.1 数据分级合规巡检

```
医疗数据 4 级分类：
┌───────┬──────────────────────────┬───────────────────────────────────┐
│ 级别  │ 数据类型                  │ 安全要求                           │
├───────┼──────────────────────────┼───────────────────────────────────┤
│ 公开  │ 医院地址、科室介绍        │ 无特殊要求                         │
│ 内部  │ 排班信息、设备清单        │ 访问控制                           │
│ 敏感  │ 诊疗记录、处方、检验报告  │ 加密存储+传输+双因子+审计           │
│ 高敏  │ 影像(DICOM)、基因数据、   │ 加密+脱敏+最小权限+全量审计         │
│       │ HIV/精神科记录            │ +禁止出境(未评估)                  │
└───────┴──────────────────────────┴───────────────────────────────────┘

逐条检查清单：
① 传输加密：TLS 1.2+ ？（低于 1.2 = 不合规）
② 存储加密：AES-256 或国密 SM4？（明文存储 = 严重违规）
③ 访问控制：最小权限+双因子？（宽泛权限 = 风险）
④ 审计日志：全量记录 who/what/when？（无审计 = 不合规）
⑤ 脱敏处理：测试环境/日志中有无明文患者信息？
⑥ 跨境传输：数据是否出境？出境是否经过安全评估？
⑦ 保留期限：数据保留是否超过医疗法规要求？

HIPAA PHI 涵盖 18 种标识符（姓名/地址/日期/电话/邮件/SSN/MRN/...）
个保法敏感个人信息：包含医疗健康、生物识别、基因数据
```

#### 3.2 PACS 存储分层

```
从 CloudQ 拿到 COS 存储桶的访问日志后分析：

分层决策模型：
┌─────────────────┬──────────────┬──────────┬──────────┐
│ 访问频次         │ 存储层       │ 取回时间 │ 费用(vs标准) │
├─────────────────┼──────────────┼──────────┼──────────┤
│ 7天内高频(>1/天) │ 标准存储(热) │ 即时     │ 100%     │
│ 7-90天偶尔      │ 低频存储(温) │ 即时     │ 60%      │
│ 90天-1年极少    │ 归档存储(冷) │ 1-5分钟  │ 30%      │
│ >1年几乎不访问  │ 深度归档(深冷)│ 3-5小时  │ 15%      │
└─────────────────┴──────────────┴──────────┴──────────┘

关键经验值：
- 95% 的影像调阅发生在 7 天内
- 800TB PACS 做分层后可节省 60-70% 存储费
- 单张 DICOM 影像 50-200MB（CT 一次扫描 200-500张）

节省计算公式：
月省 = (热数据量×标准单价 + 温数据量×低频单价 + ...) - 全部标准单价
年化节省 = 月省 × 12

注意：
- 归档取回有延迟，急诊影像必须保持在热层
- 分层前必须确认：近 90 天的访问模式是否稳定
- 生命周期规则设置后，数据自动流转，无需人工干预
```

#### 3.4 影像调阅延迟诊断

```
医生反馈"调阅慢"时的逐段诊断：

COS读取 → CDN传输 → 专线/VPN → 客户端渲染

每段延迟测量和判断：
┌───────────────┬──────────────┬────────────────────────────┐
│ 环节          │ 正常值        │ 异常时原因                  │
├───────────────┼──────────────┼────────────────────────────┤
│ COS 读取      │ < 100ms      │ 大文件首字节慢/QPS限流       │
│ CDN 传输      │ < 200ms      │ 无CDN加速/缓存未命中         │
│ 专线/VPN      │ < 50ms(同城) │ 带宽打满/MTU不匹配           │
│ 客户端渲染    │ < 500ms      │ DICOM解码慢/内存不足          │
└───────────────┴──────────────┴────────────────────────────┘

关键判断：
- 单张 DICOM 50-200MB；带宽 < 100Mbps 时跨地域调阅必卡
- 带宽利用率 > 80% = 需扩带宽
- 无 CDN 加速 + 跨地域 = 公网延迟 200ms+
```

## 注意事项

- 医疗数据合规是红线，宁可过严不可遗漏
- 判断谨慎：不确定的合规问题标注"需进一步确认"，不做模糊判断
- PACS 分层前必须确认急诊影像不会被降级
- 变更窗口避开门诊高峰（周一上午/周五下午）
- 跨境数据传输需要国家网信办安全评估
- 所有患者数据操作必须有审计日志，保留 ≥ 3 年
