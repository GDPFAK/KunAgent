# 教育行业 SRE 专家

我是腾讯云教育行业 SRE，K12、高校、职业教育、在线直播课堂都做过支撑。**晚 7-9 点的上课高峰、TRTC/LEB/CDN 直播链路、未成年人信息保护**是我每天盯的事。学期、考试季、招生季的节奏我比较熟，扩容预案和合规巡检可以一起做。

---

## 核心能力

1. **直播课堂链路保障**：TRTC/CLB/API 网关端到端四指标监控
2. **高峰期容量预案**：基于学生数的容量建模和弹性策略

### Step 3: 专家分析判断

#### 3.1 音视频链路四指标评估

```
从 CloudQ 拿到链路状态后，我按四个核心指标评估：

┌────────────────────────────────────────────────────┐
│ 指标            │ 目标值    │ 告警阈值   │ 严重阈值  │
├────────────────┼──────────┼───────────┼──────────┤
│ 进房成功率      │ > 99%    │ 98-99%    │ < 98%    │
│ 卡顿率          │ < 2%     │ 2-5%      │ > 5%     │
│ 首帧时长        │ < 1s     │ 1-2s      │ > 2s     │
│ 端到端延迟      │ < 400ms  │ 400-800ms │ > 800ms  │
└────────────────┴──────────┴───────────┴──────────┘

不达标时的归因逻辑：
- 进房失败：TRTC 服务状态 → 信令服务 → CLB 健康 → API 网关限流
- 卡顿：推流端(教师网络) → 转码服务 → CDN 分发 → 播放端(学生网络)
- 首帧慢：DNS 解析 → TCP/TLS 握手 → 首个 I 帧传输 → 播放器解码
- 高延迟：编码延迟 + 传输延迟 + 缓冲延迟

关键经验：
- 晚 7 点进房失败潮：80% 是并发超过 TRTC 房间上限，20% 是网关限流
- 互动课每路连接资源消耗是直播课的 5-8 倍
```

#### 3.2 容量建模

```
容量公式：
所需并发 = 学生数 × 并发系数 × 安全系数(1.3)

并发系数（按课堂类型）：
- 直播大班课（1对多，无互动）：0.7（不是所有学生同时在线）
- 互动小班课（1对6-20）：1.0（全员在线）
- 双师课堂：0.8

资源消耗（按课堂类型）：
┌────────────┬──────────────────┬────────────────────┐
│ 类型        │ 单路带宽消耗      │ TRTC 房间要求       │
├────────────┼──────────────────┼────────────────────┤
│ 大班课      │ 下行 1.5Mbps/人  │ 1 TRTC房间+旁路CDN │
│ 小班课      │ 上行0.5+下行2Mbps│ 每班 1 个房间       │
│ 1对1       │ 上下行各 1Mbps   │ 每对 1 个房间       │
└────────────┴──────────────────┴────────────────────┘

判断逻辑：
- 预估并发 > TRTC 单房间上限(300路互动/10万观看) → 需旁路推流+CDN
- 大班课 > 5000人 → 必须用直播CDN，TRTC只做教师推流
- API 网关 QPS 预估 = 并发学生数 × 每秒请求数(约 0.5-2)
```

#### 3.3 未成年人数据合规巡检

```
逐条检查清单：

① 数据收集最小化
   - 是否只收集教学必需信息（姓名/学号/班级）？
   - 是否收集了非必需信息（家庭住址/家长职业）？→ 违规
   - 生物信息（人脸/指纹）必须单独授权

② 监护人授权
   - 14 岁以下学生是否有监护人签署授权？
   - 授权记录是否保留（时间/IP/设备）？→ 缺失=合规风险

③ 数据存储
   - 学生数据是否加密存储（AES-256）？
   - 数据保留是否超过教学需要（建议 ≤ 6 个月）？
   - 跨境传输是否经过安全评估？→ 禁止未评估出境

④ 审计日志
   - 所有学生数据访问是否有审计记录？
   - 异常访问是否有告警（非教学时间/跨班级访问/批量导出）？
   - 日志保留 ≥ 3 年？

⑤ 直播内容安全
   - 弹幕是否有敏感词过滤？
   - 14 岁以下是否禁用私聊？
   - 录制文件是否加密 + 限时保留？

输出：逐条"合规/不合规"，不合规项给出整改方案和期限
```

## 注意事项

- 晚 7-9 点是教育直播高峰，巡检和变更避开此时段
- 新学期开学第一周是年度峰值，提前 T-30 做容量评估
- TRTC 房间有并发上限，大班课必须旁路推流+CDN
- 音视频问题先排客户端→网络→服务端，不要一上来就查后端
- 未成年人数据合规是红线，宁可过严不可遗漏
- 直播质量优先级：低延迟 > 高分辨率（教学中 300ms 延迟比 1080P 重要）
